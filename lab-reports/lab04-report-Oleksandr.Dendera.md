Хід роботи
1. Обґрунтування вибору СУБД та структури даних Для реалізації інтернет-магазину кросівок було обрано вбудовану реляційну базу даних SQLite. Це дозволяє зберігати дані у локальному файлі без необхідності налаштування окремого сервера БД, що є оптимальним для навчального проекту та контейнеризації.

Було спроектовано таблицю products для зберігання інформації про товари. Основні поля:

id (Integer, Primary Key) — унікальний ідентифікатор;

name (Text) — назва кросівок;

price (Real) — ціна;

image (Text) — посилання на зображення;

category (Text) — категорія (Running, Casual, Luxury);

description (Text) — опис товару.

2. Реалізація підключення до бази даних У файлі models.py було створено функцію get_db_connection, яка встановлює з’єднання з файлом бази даних та налаштовує формат отримання даних (як словник sqlite3.Row).

Лістинг коду (підключення):

Python

import sqlite3
import os

DB_PATH = os.environ.get('DATABASE_PATH', 'db.sqlite')

def get_db_connection():
    conn = sqlite3.connect(DB_PATH)
    conn.row_factory = sqlite3.Row
    return conn
3. Створення таблиць (Ініціалізація БД) Реалізовано функцію init_db, яка при запуску перевіряє наявність таблиць і створює їх за допомогою SQL-запиту CREATE TABLE IF NOT EXISTS.

Лістинг коду (створення таблиці товарів):

Python

def init_db():
    conn = get_db_connection()
    c = conn.cursor()
    
    # Таблиця товарів
    c.execute('''
        CREATE TABLE IF NOT EXISTS products (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            name TEXT NOT NULL,
            price REAL NOT NULL,
            image TEXT,
            category TEXT,
            brand TEXT,
            description TEXT
        )
    ''')
    conn.commit()
    conn.close()
